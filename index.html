Why story mapping?
Because written requirements alone are a terrible way to build ANYTHING!
Don't believe ME?
Write Happy 40th Birthday Kim! "Make 40 bigger"
Still don't believe ME?
Make the cake look just like the picture with "Happy Birthday Tayden" at the bottom and blue balloons around the edge
Also because flat backlogs are terrible at communicating the big picture.
How does a story map help?

Everybody can see the big picture

Create from broad to small

Keeps the focus on end user needs

Organize roadmaps, releases, &
minimal viable products

Discover holes in the user workflows

Discover feature waste

Better way to discover business value

Gets the team & shareholders talking & collabrating

How do you create a story map?

Create user personas

Indentify the user goals

Determine the user activities that can achieve those goals

Ascertain the user tasks that will complete those activities

Organize workflows into release groups

Use paper, markers, printouts, drawings, & pictures

All of this should be created from lots and lots of conversations

Drawing of the backbone
Drawing adding the tasks
story map release feeds into backlog
Team working on story map
Final story map
